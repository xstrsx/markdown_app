# WebDAV 云同步设计

## 目标

在现有 Markdown 编辑器中增加可选的 WebDAV 云端文件流。未配置 WebDAV 时，首页新建/打开、编辑器保存、另存为和历史记录保持现有本地行为；配置生效后，涉及文件来源选择的入口允许用户选择本地或云端，云端打开的文档保存时直接覆盖原远程文件。

## 已确认的边界

- WebDAV 使用配置的远程根目录作为沙盒，云端选择器支持进入子目录和返回上级。
- 云端另存为在当前目录输入文件名，不改变当前选择器目录。
- WebDAV 密码使用 `flutter_secure_storage` 保存；服务器地址、用户名、远程根目录和启用状态使用现有 `shared_preferences` 保存。
- 云端请求失败只显示错误，不自动切换到本地，也不把云端文档静默另存到本地。
- 云端打开的文件手动保存、自动保存和退出保存均直接覆盖其远程路径。
- 本地文件继续使用现有 `FileService` 和 Android `content://` 流程。

## 依赖与服务边界

采用 `webdav_client: ^1.2.2`。该版本提供 `newClient`、`ping`、`readDir`、`read`、`write(Uint8List)` 和 `mkdirAll`，并负责 Basic/Digest 认证及 WebDAV XML 解析。

新增 `WebDavService`，不让页面直接依赖第三方 `Client`。服务负责：

- 根据配置创建客户端并设置连接、发送和接收超时。
- 将远程根目录规范化为以 `/` 开头、无越界 `..` 的路径。
- 读取当前目录并转换为项目自己的远程条目模型。
- 下载 UTF-8 Markdown 字节、上传 UTF-8 内容、测试连接。
- 保证选择器只能访问配置根目录及其子目录。
- 将第三方异常转换为带用户可读信息的项目异常，并保留 `debugPrint` 日志。

新增 `WebDavGateway` 抽象作为测试替身边界；生产实现包装 `webdav_client.Client`，单元测试使用内存 fake 验证目录、读取和写入行为，不发真实网络请求。

## 设置模型与持久化

扩展 `AppSettings`，增加 `WebDavConfig`：

- `enabled`
- `serverUrl`
- `username`
- `rootPath`
- `password`（仅运行时存在，不写入 SharedPreferences）

配置完整条件为启用、服务器 URL 合法、根目录有效；用户名和密码允许为空，以兼容匿名 WebDAV。设置页提供启用开关、服务器地址、用户名、密码、远程根目录和“测试连接”操作。保存配置后立即更新根状态；测试连接只在完整配置下执行，失败保留表单并显示错误。

`SettingsService` 继续负责普通偏好，并通过可注入的安全存储接口读写密码。读取任意设置失败时回退到安全默认值并记录日志；保存失败不让界面崩溃，保留用户当前内存设置。

## 文件来源与历史记录

扩展 `MarkdownFile` 增加来源信息：

- `storageType`: `local` 或 `webDav`
- `remotePath`: 云端文件在 WebDAV 根目录下的规范化路径

旧 JSON 缺少来源字段时按本地文件解析。云端文件仍把下载内容缓存到本地 `contentPath`，以便历史列表显示和离线查看，但所有写入行为依据 `storageType` 分流到 WebDAV，不依据缓存路径误写本地。

历史记录序列化保存来源和远程路径；云端记录重新打开时使用当前 WebDAV 配置下载，配置缺失或请求失败只提示错误，不删除历史记录。

## 用户流程

### 配置未生效

所有现有入口不出现云端选项：

- 首页新建继续创建本地文件。
- 首页打开继续使用系统文件选择器。
- 编辑页另存为继续使用现有本地保存对话框。

### 配置生效

首页新建、首页打开和编辑页另存为先显示“本地 / 云端”选择对话框。

- 首页新建选择云端：打开云端保存选择器，选择目录并输入 Markdown 文件名；编辑器以云端新文件元数据打开，首次保存时上传。
- 首页打开选择云端：打开云端文件选择器，只显示文件夹和 Markdown 文件；选择文件后下载并进入编辑器。
- 编辑页另存为选择云端：从当前目录打开云端保存选择器，完成后把当前文档切换为新的云端远程路径。
- 已打开云端文档的保存和自动保存不再弹来源选择，直接 PUT 覆盖 `remotePath`。

云端选择器显示当前路径、返回上级、刷新、进入文件夹和选择文件；任何请求失败都停留在当前流程并显示错误。

## 编辑器保存分流

保留现有本地保存快照竞态保护。`EditorPage` 的保存入口统一先判断当前文件来源：

- 本地：调用现有 `_saveFile` 本地流程。
- 云端：使用当前文本 UTF-8 编码调用 `WebDavService.writeFile(remotePath, bytes)`，成功后更新缓存、历史记录和修改状态。

保存失败时保留 `_isModified = true`，不更新历史记录中的云端内容，不关闭编辑器。自动保存使用相同分流函数，因此不会产生“自动保存到本地缓存但云端未更新”的假成功。

## 错误与安全

- 配置表单拒绝空 URL、非法 URL、空根目录和包含根目录外跳转的路径。
- WebDAV 请求设置有限连接/发送/接收超时；不实现后台无限重试。
- 用户名、密码不写入日志；第三方异常日志需过滤可能包含认证 URL 的内容。
- 云端失败不降级到本地，避免用户误以为云端文件已保存。
- 删除、重命名和批量同步不在本次范围内。

## 测试策略

- 设置服务：普通配置持久化、安全密码读写、旧配置兼容、非法配置回退和保存异常。
- WebDAV 服务：路径规范化和根目录越界拒绝、目录列表过滤 Markdown、下载/上传、网络异常转换；使用 fake gateway。
- 设置页：无配置时表单显示、启用/禁用、连接测试反馈、密码字段不明文展示。
- 选择器：目录进入、返回上级、Markdown 文件过滤、保存文件名校验和请求失败提示。
- 编辑器：云端文件加载、手动保存覆盖远程路径、自动保存调用同一远程分流、失败保持 dirty；保留现有本地保存和导航测试。
- 最终运行 `dart format`、`flutter analyze --no-pub` 和完整 `flutter test --no-pub`。
